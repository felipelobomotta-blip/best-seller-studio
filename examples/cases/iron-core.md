# Case 7: Iron Core

## Snapshot

- Genre: LitRPG / dungeon core / progression fantasy
- Language: EN
- Target length: 120k words
- Planned chapters: 34
- Pipeline: V5 production
- Public manuscript: no

## Custom Scoring Dimension

```text
SRE-Methodology Consistency
```

This measures whether the protagonist thinks like a site reliability engineer rather than a generic modern-world tinkerer.

## Why It Matters

Iron Core tests whether Book Genesis can build a commercially legible genre book around a narrow, defensible professional edge: incident reviews, blameless postmortems, versioned traps, A/B tested encounters, runbooks, SLOs, and error budgets translated into dungeon-core fantasy.

## What The Pipeline Had To Control

- the protagonist's edge had to be process, not modern artifacts
- the healer character had to be an auditor, not a reward
- progression needed to satisfy LitRPG readers without turning into a spreadsheet
- the book needed enough audiobook-friendly prose for the niche

## What It Taught The System

Custom dimensions work best when they protect the book's market wedge. For Iron Core, the differentiator is not "dungeon core with tech," but "dungeon core with operational methodology."
