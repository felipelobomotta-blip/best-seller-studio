Post this. No link in the first paragraph. Reply to every comment for the first 6 hours.

**Title:**
```
I built a Claude Code skill suite that runs a full AI book studio — here is what I learned about agent orchestration
```

**Body:**
```
I kept hitting the same problem with AI-assisted fiction: scoring during drafting makes the draft worse. The writer starts optimizing for the evaluator instead of the story.

So I rebuilt the system as Best Seller Studio — a file-backed skill suite that can run in Claude Code, Codex, Kimi, Antigravity, or basically any capable agent that can read/write files.

The key design changes:
- one active phase prompt at a time
- PROJECT_STATE.yaml and ASSUMPTIONS.md persist state
- draft before judgment
- adversarial audit before final score
- 10-dimension Genesis Score
- editorial package at the end

The surprising lesson: the 17-phase, 19-skill version caught more errors but made the books worse. The writer learned it was being judged on too many dimensions simultaneously and stopped writing like a writer.

Proof layer: 10 book projects in under 30 days — memoir, fantasy, thriller, sci-fi, LitRPG, cozy mystery, dark academia. The manuscripts are private; the repo publishes the pipeline, covers/cover concepts, case studies, scoring artifacts, outlines and synopses.

Repo: https://github.com/felipelobomotta-blip/best-seller-studio

Curious if others building long-form agent workflows have hit the same tradeoff: more orchestration improves consistency but too many live constraints hurt voice.
```
